Getting started

Introduction to Space

Space is a minimal, compiled programming language that translates directly to x86-64 assembly. It has three keywords, two data types, and zero runtime dependencies. If you know what a variable is, you can already read Space code.

Space was designed around one idea: the gap between writing code and running real machine instructions should be as small as possible. When you compile a Space program, you get a native Linux binary — no virtual machine, no garbage collector, nothing in between.

The language is intentionally small. There are no imports, no functions, no control flow — yet. Version 1.0 is the foundation. Every program is a flat sequence of statements that declare variables, print output, and exit.

Space v1.0 — This is the first public release. The language is actively growing. Core syntax documented here is stable.

Installation

Space currently runs on Linux x86-64. The project ships with an installer script that builds the compiler from source and registers a global space command. Here's the complete workflow, start to finish.

1. Prepare the folder

Download or clone the project folder. It should contain src/, libs/, CMakeLists.txt, and install.sh.

Open your file manager, right-click inside that main project folder, and select "Open in Terminal".

2. Make the installer executable

Before Linux lets you run a script, you have to give it permission:

# grant execute permission
chmod +x install.sh

3. Run the installation script

Execute the script to install dependencies, build your C code, and place the space command in your global system path:

./install.sh

This will likely ask for your sudo password — it installs NASM/CMake and moves the compiled binary to /usr/local/bin/.

4. Write your code

Once installed, you can close that terminal. Open a new terminal in any other folder — your Desktop, Documents, wherever your project lives.

Create your file:

# create a new Space file
nano try.sp

Add your syntax:

planet x = 10;
emit x;

Save and close the file.

5. Run the compiler

Execute your file using the globally installed command:

space try.sp

What happens behind the scenes: the compiler reads try.sp, generates code.asm in that same folder, uses NASM to assemble it into code.o, links it into an executable named code, and immediately runs it — printing 10 to your screen.

Your first program

Create a file named hello.sp and write the following:

planet greeting = "hello from space";
dock greeting;
emit 0;

Once the compiler is installed, compile and run it:

# compile
./boom hello.sp

# this produces and runs a native binary
# output: hello from space

That's it. Three lines, three keywords, one native binary.

Keywords

Space has exactly three reserved keywords. Every program is built from these three primitives.

planet
Declare and assign a variable
dock
Print a value to stdout
emit
Exit the program with a code

These keywords cannot be used as variable names. Everything else — identifiers, literals, operators — is fair game.

Variables

Variables are declared with the planet keyword, followed by a name, an = sign, a value, and a semicolon.

planet name = value;

Integer variable

planet x = 42;
planet count = 100;

String variable

planet msg = "hello world";
planet lang = "space";

Variable from expression

planet a = 10;
planet b = a * 2 + 5;
planet c = a + b;

Variable names must start with a letter and contain only letters and digits. Underscores are not currently supported.

Types

Space v1.0 has two data types. The type of a variable is inferred from its value at declaration — there is no type annotation syntax.

Integer

A whole number. Stored as a 64-bit signed integer on the stack. Supports all arithmetic operators.

planet x = 99;

String

A sequence of characters enclosed in double quotes. Stored as a pointer to a null-terminated byte string in the .data section.

planet name = "space";

Strings cannot currently be used in arithmetic expressions. Arithmetic is integers-only.

Operators

Space supports five arithmetic operators. They follow standard mathematical precedence — multiplication, division, and modulo are evaluated before addition and subtraction.

OperatorNameExampleResult
+Addition3 + 47
-Subtraction10 - 37
*Multiplication3 * 412
/Division10 / 25
%Modulo10 % 31

Precedence

Operators are evaluated in this order (highest to lowest):

  • * / % — evaluated first
  • + - — evaluated after
// evaluates as (2 * 3) + (4 / 2) = 8
planet x = 2 * 3 + 4 / 2;

// chain variables in expressions
planet a = 5;
planet b = a * a - 1;

Output

The dock keyword prints a value to standard output, followed by a newline. It accepts integer literals, string literals, or variable names.

Print a string literal

dock "hello world";

Print an integer literal

dock 42;

Print a variable

planet score = 100;
dock score;

When printing an integer, Space converts the number to its decimal digit representation at runtime using a divide-loop — no sprintf, no libc. When printing a string, it computes the length at runtime and makes a single sys_write syscall.

Exit codes

Every Space program should end with an emit statement. This maps directly to the Linux sys_exit syscall.

emit 0;   // success
emit 1;   // general error

By convention, exit code 0 means the program completed successfully. Any non-zero value signals an error. The value is passed directly to the OS — you can check it with echo $? in your shell after running the binary.

emit also accepts an expression: emit x + 1; — whatever the expression evaluates to becomes the exit code.

Example — Hello world

The simplest possible Space program.

dock "hello, world";
emit 0;

Example — Arithmetic

Storing values, computing expressions, and printing results.

planet a = 12;
planet b = 4;

planet sum  = a + b;
planet diff = a - b;
planet prod = a * b;
planet quot = a / b;
planet rem  = a % b;

dock sum;   // 16
dock diff;  // 8
dock prod;  // 48
dock quot;  // 3
dock rem;   // 0

emit 0;

Example — Working with strings

Storing text and printing it alongside numbers.

planet lang    = "space";
planet version = 1;

dock "language:";
dock lang;

dock "version:";
dock version;

emit 0;

How the compiler works

The Space compiler is a single-pass C program that walks through five stages:

  • Lexer — reads your source file character by character and produces a flat list of tokens (keywords, identifiers, literals, operators, semicolons).
  • Parser — walks the token list and builds an Abstract Syntax Tree (AST) using recursive descent. Operator precedence is enforced here.
  • AST walker — traverses the tree node by node and emits NASM assembly instructions for each construct.
  • Assembler — NASM converts the generated .asm file into an object file.
  • Linker — ld links the object file into a standalone native binary.

Variables are stored on the stack relative to rbp. The compiler tracks each variable's stack offset and type so dock knows whether to run the integer digit-split routine or the string length counter.

File conventions

Space source files use the .sp extension.

hello.sp
myprogram.sp

The compiler is invoked as:

./boom <file>.sp

It will produce code.asm, code.o, and a final executable called code in the current directory, then run it immediately.

Output binary naming and directory control will be added in a future release.
